FbPlaybook.js
Create, save, and view fluid American Football play animations!
Documentation
The first functionality provided by this library is the ability to create Playbooks, that the end user can flip through and look at. The developer can provide a name, field object, a boolean to signify if the user can save plays, and an id for an html element for which the object will be inserted into. Since we are creating a standalone Playbook, we set the field argument to null and the boolean to false. The book starts with no pages, so the user is unable to flip at this point:
const book = new playBook("Sample", null, false, someid);
You can now add plays as such. Below is an example of how to add one page, however this book below has four:
book.addPage( "https://bestyouthfootballplays.com/wp-content/uploads/10-QB-Sneak-I-630x512.png", "QB Sneak", "https://youtu.be/qyqCTMirNWg?t=86");
Now for the interesting part. The devloper can also create PlayDisplayer objects which can showcase animations. It takes in a size, name, and id of the element for which it should be inserted. When the displayer is initially created, there will be no animation:
const field = new PlayDisplayer('large', 'Offence', someid);
The developer can now set a default play. They do this setting the move for each position manually, using a list of predefined legal moves. See documentation for the list of legal moves. It is important that every field has a different name, so this one is called Offence2 now.
field.setLTEMove('deep-90-right');
field.setRTEMove('mid-90-left');
field.setQBMove('pass-qb');
field.setRTMove('short-90-right');
field.setFBMove('hole-five-fb');
The developer can also create a code sandbox for the end user to play with. We have not connected a field to a playBook yet, so all the end user can do is play around with animations; they are not able to save any in this case. We call this method with null because that is a boolean to allow saving plays.
field.spawnSandbox(null, someid);
We can also create and connect a playbook to the play displayer, by creating a new playbook and passing the play displayer object as an attribute. The developer can then add plays to the playbook for the end user to view the animations. The new attribute to the addPage method is a list of legal move names for each player, in order of the formation left to right (LTE, LT LG, C, RB, RT, RTE, QB, LHB, FB, RHB). Flip to the play you want to see, click Initialize Play, then click Play Animation:
const field2 = new PlayDisplayer('large', 'Connected', someid);
const connectedbook = new playBook("Connected", field4, false, someid);
connectedbook.addPage("https://i.ibb.co/kSFmpZV/Hail-Mary-Out.png", "Hail Mary Out","https://youtu.be/qyqCTMirNWg?t=289", ['straight-deep', 'mid-90-left', 'none', 'none', 'none', 'mid-90-right', 'straight-deep', 'pass-qb', 'none', 'hole-four-fb', 'none'] );
We can also allow the user to create plays without images or links and add them to the playbook, we do this by signifying this by setting the boolean to true when we create the book before adding the pages. We also call spawnSandbox with the boolean set to true to allow the user to change the name of the play before saving.
const connectedbook2 = new playBook("Connected", field4, true, someid);
connectedbook.addPage("https://i.ibb.co/kSFmpZV/Hail-Mary-Out.png", "Hail Mary Out","https://youtu.be/qyqCTMirNWg?t=289", ['straight-deep', 'mid-90-left', 'none', 'none', 'none', 'mid-90-right', 'straight-deep', 'pass-qb', 'none', 'hole-four-fb', 'none'] );
field2.spawnSandbox(true, someid)

connectedbook2.allowUserCreatePlays(someid)
